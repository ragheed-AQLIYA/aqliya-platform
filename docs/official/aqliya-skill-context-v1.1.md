# AQLIYA Skill Context v1.1

**Purpose:** Defines the documentation, architecture, governance, and truthfulness skills required for work on AQLIYA.  
**Version:** 1.1  
**Status:** Official

---

## Skill 1: Platform Identity Awareness

- Never reduce AQLIYA to AuditOS only.
- Never describe AQLIYA as SaaS-only.
- Keep Cloud active and Private strategic.
- Keep products, shared applications, custom workspaces, demos, and prototypes clearly separated.

## Skill 2: Reality Classification Discipline

Every implemented area must be classified using the official release model:

- Included in v0.1
- Included as pilot-ready product
- Included as active adjacent system
- Included as governed shared application
- Included as custom/internal workspace
- Included as demo only
- Prototype / internal preview
- Strategic / future
- Not implemented
- Do not claim as live

## Skill 3: Governance-First Framing

- Every real system must be described with permission, review, evidence, and audit-trail boundaries.
- AI must remain assistive, reviewable, and non-autonomous.
- Sensitive internal or custom surfaces must not be overpromoted into general product claims.

## Skill 4: Route Discipline

| Need                       | Route Family                              | Rule                                                |
| -------------------------- | ----------------------------------------- | --------------------------------------------------- |
| Marketing                  | `/products/*`, public pages               | Describe, do not overclaim                          |
| Governed product workspace | `/audit/*`, `/decisions/*`                | Real product surfaces                               |
| Shared application         | `/assistant/*`                            | Governed shared application, not standalone product |
| Custom workspace           | `/sunbul/*`, `/workflowos/*`              | Real but custom/internal classification             |
| Demo                       | `/auditos/*`                              | Demo only                                           |
| Prototype preview          | `/sales`, `/organizations/*`, `/settings` | Must be labeled honestly                            |

## Skill 5: Documentation Synchronization

When system reality changes, update:

- official docs
- source-of-truth docs
- release-scope docs
- demo/commercial safety docs

Do not leave official docs behind source-of-truth when the repository reality is already hardened.

## Skill 6: Commercial Truthfulness

Allowed examples:

- AuditOS is real and pilot-ready.
- DecisionOS is real and active.
- Office AI Assistant is a governed shared application.
- Sunbul is a custom/client-specific workspace.

Forbidden examples:

- SalesOS is live.
- LocalContentOS workspace is implemented.
- AQLIYA has a production On-Prem package.
- AQLIYA has Local AI runtime, Studio, Model Governance, or Institutional Memory live.
