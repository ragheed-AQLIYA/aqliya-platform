# AuditOS — قائمة تهيئة العميل للبايلوت

## المرحلة ١: ما قبل البدء (قبل البايلوت بـ ٥ أيام عمل)

### إداري وقانوني
- [ ] توقيع اتفاقية سرية وعدم إفصاح (NDA)
- [ ] توقيع اتفاقية نطاق البايلوت
- [ ] توقيع اتفاقية استخدام البيانات لاغراض البايلوت فقط
- [ ] تحديد جهة اتصال رئيسية من طرف العميل (Project Lead)
- [ ] تحديد الأدوار: مدقق، مراجع أول، شريك

### تقني
- [ ] التأكد من توفر المتطلبات التقنية (متصفح حديث، اتصال إنترنت مستقر)
- [ ] إرسال دعوات إنشاء الحسابات للمستخدمين
- [ ] تفعيل الحسابات من قبل كل مستخدم
- [ ] اختبار الدخول للنظام لجميع المستخدمين

### ملفات
- [ ] استلام ملف ميزان المراجعة والتأكد من صيغته
- [ ] استلام دليل الحسابات (إن وجد)
- [ ] استلام القوائم المالية للعام السابق (إن وجدت)
- [ ] استلام معلومات المنشأة الأساسية
- [ ] التحقق من اكتمال البيانات المطلوبة (راجع قائمة الملفات المطلوبة)

## المرحلة ٢: التدريب الأولي (قبل البايلوت بـ ٣ أيام عمل)

- [ ] جلسة تدريب: نظرة عامة على AuditOS وسير العمل (ساعة)
- [ ] جلسة تدريب: تطبيق عملي على بيانات تجريبية (ساعة)
- [ ] شرح الأدوار والصلاحيات لكل مستخدم
- [ ] شرح حدود النظام المعروفة (Known Limitations)
- [ ] شرح قنوات الدعم والتواصل أثناء البايلوت
- [ ] توزيع دليل المستخدم السريع (Quick Reference Guide)
- [ ] اختبار فهم الفريق: تشغيل سيناريو بسيط

---

## المرحلة ٣: إعداد البيئة (قبل البايلوت بـ يوم عمل)

### فني — من طرف AQLIYA
- [ ] إنشاء نسخة احتياطية أولية من قاعدة البيانات
- [ ] تشغيل فحص السلامة (Health Check): 7/7
- [ ] إنشاء منظمة العميل (Organization) في النظام
- [ ] إنشاء حسابات المستخدمين بالأدوار المحددة
- [ ] التحقق من صلاحيات كل مستخدم
- [ ] إنشاء المهمة (Engagement) الأساسية
- [ ] رفع ميزان المراجعة والتأكد من قراءته
- [ ] التحقق من سجل التدقيق (Audit Trail) يعمل

### عميل — من طرف العميل
- [ ] التأكد من دخول جميع المستخدمين للنظام
- [ ] التأكد من ظهور المهمة (Engagement) في لوحة التحكم
- [ ] مراجعة البيانات المرفوعة والتأكد من صحتها
- [ ] إبلاغ AQLIYA بأي ملاحظات قبل البدء الرسمي

---

## المرحلة ٤: يوم الانطلاق (Day 0)

- [ ] جلسة افتتاحية (٣٠ دقيقة): تأكيد الجدول وقنوات التواصل
- [ ] بدء العمل على مرحلة توجيه الحسابات (Mapping)
- [ ] متابعة فورية لأي مشكلة تقنية خلال الساعتين الأولى
- [ ] تسجيل الملاحظات الأولية من الفريق

## قنوات التواصل أثناء البايلوت

| القناة | الاستخدام |
|--------|-----------|
| WhatsApp | استفسارات سريعة، دعم يومي |
| البريد الإلكتروني | توثيق الملاحظات الرسمية وتقارير التقدم |
| جلسات أسبوعية | متابعة التقدم، استعراض التحديات، جمع التغذية الراجعة |
| جلسة طارئة | عند الحاجة، تُعقد خلال ٢٤ ساعة من الطلب |

## أدوار الفريق

| الدور | المسؤولية أثناء التهيئة |
|-------|------------------------|
| **قائد المشروع (عميل)** | تنسيق التسليم، متابعة الفريق، نقطة اتصال رئيسية |
| **مدير البايلوت (AQLIYA)** | إعداد البيئة، التدريب، الدعم الفني، توثيق النتائج |
| **المستخدمون (عميل)** | إكمال التدريب، استخدام النظام، تقديم التغذية الراجعة |
