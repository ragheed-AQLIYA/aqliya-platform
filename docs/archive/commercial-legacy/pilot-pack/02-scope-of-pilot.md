# AuditOS — نطاق البايلوت

## الهدف من البايلوت
اختبار AuditOS على مهمة تدقيق حقيقية واحدة للتحقق من:
1. اكتمال دورة العمل بالكامل ببيانات حقيقية
2. صحة المخرجات (القوائم، الإيضاحات، الملاحظات)
3. صلاحية النظام للاستخدام المؤسسي
4. جاهزية العميل للانتقال إلى الاشتراك التجاري

## داخل النطاق (In Scope)

### المهمة (Engagement)
- مهمة تدقيق واحدة (أو مهمة إعداد قوائم) لشركة حقيقية
- فترة مالية واحدة (سنة)
- ميزان مراجعة فعلي يحتوي جميع الحسابات

### مراحل العمل المشمولة
| المرحلة | الوصف |
|---------|-------|
| ١. رفع ميزان المراجعة | رفع ملف CSV/XLSX وتحقق النظام من اتزانه |
| ٢. توجيه الحسابات (Mapping) | اقتراح الذكاء الاصطناعي لتبويب الحسابات + مراجعة بشرية |
| ٣. مراجعة ميزان المراجعة | فحص العلاقات المحاسبية واكتشاف الشذوذ |
| ٤. مسودة القوائم المالية | إنشاء قائمة المركز المالي، قائمة الدخل، قائمة التغيرات في حقوق الملكية |
| ٥. مسودة الإيضاحات | إنشاء الإيضاحات المطلوبة مع تحديد النواقص |
| ٦. طلبات الأدلة | تحديد الأدلة الداعمة المطلوبة لكل منطقة حسابية |
| ٧. ملاحظات المراجعة | توليد ملاحظات بتصنيف الخطورة وربطها بالأدلة |
| ٨. المراجعة والاعتماد | مسار المراجعة (مراجع → شريك) مع تسجيل القرارات |
| ٩. المصادقة (Validation) | تشغيل فحوصات النظام وتسوية الملاحظات |
| ١٠. النشر | إصدار حزمة التعامل مع سجل التدقيق الكامل |

### الأدوار المشمولة
- **مدقق / معدّ** (Operator): رفع البيانات، توجيه الحسابات، رفع الأدلة
- **مراجع أول** (Reviewer): مراجعة الملاحظات، إبداء الرأي، تسوية المصادقة
- **شريك** (Partner): الاعتماد النهائي والنشر

### الدعم الفني
- جلستان أسبوعياً (ساعة لكل جلسة)
- قناة تواصل مباشر (WhatsApp)
- تدريب أولي (ساعتان) قبل بدء العمل

## خارج النطاق (Out of Scope)

| البند | السبب |
|-------|-------|
| تصدير PDF / DOCX / XLSX | النسخة الحالية تدعم JSON فقط |
| الدخول الموحّد (SSO / OAuth) | قيد التطوير — الدخول حالياً بالبريد وكلمة المرور |
| بوابة العميل الخارجية | مرحلة لاحقة بعد الانتقال التجاري |
| أكثر من مهمة واحدة | البايلوت مخصص لمهمة واحدة فقط |
| أكثر من ٥ مستخدمين | السعة الحالية محدودة للبايلوت |
| فحص البرمجيات الخبيثة للملفات المرفوعة | غير مفعّل في بيئة التطوير |
| ربط مع أنظمة خارجية (ERP) | مرحلة لاحقة |
| تدقيق الزكاة والضرائب | خارج نطاق المنتج الحالي |

## محددات معروفة (Known Limitations)
- النسخة الحالية Pilot-ready وليست Commercial-ready
- التصدير بصيغة JSON فقط
- لا يوجد مسح آلي للبرمجيات الخبيثة في بيئة البايلوت
- الدخول بالبريد وكلمة المرور فقط
- الأدلة تخزّن كمراجع (metadata) وليس كملفات ثنائية مخزنة
- قائمة التدفقات النقدية لم تنفذ بعد
- النسخ الاحتياطي يدوي وليس آلياً
