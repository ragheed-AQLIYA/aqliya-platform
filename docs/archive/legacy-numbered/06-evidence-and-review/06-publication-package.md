# Publication Package

## Purpose

This document defines the structure and content of the AuditOS publication package — the final deliverable produced after all reviews and approvals are complete.

## Package Contents

The publication package includes:

```txt
Publication Package — Client Name — FY 2026
│
├── Cover Page
├── Draft Financial Statements
│   ├── Statement of Financial Position
│   ├── Statement of Profit or Loss
│   ├── Statement of Changes in Equity (if generated)
│   └── Statement of Cash Flows (if generated)
├── Draft Notes to Financial Statements
├── Account Mapping Summary
├── Reclassification Summary
├── Review Findings Summary
├── Approved Recommendations
├── Evidence Coverage Report
├── Missing Information Log
└── Audit Trail
```

## Draft Financial Statements

Each statement includes:

- Standard IFRS/SOCPA presentation format
- Comparative period data (where available)
- Note cross-references
- Draft marking on every page
- Rounding level as configured

## Draft Notes

Each note includes:

- Standard note format with headings
- Populated amounts from trial balance data
- Accounting policy templates
- Missing data flags (if applicable)
- Cross-references to statement line items

## Package Delivery

The publication package is:

1. Generated as a structured digital package
2. Available through the AuditOS portal for authorized client contacts
3. Exportable for integration with firm's documentation system
4. Immutable — once published, the package is versioned and preserved

## Post-Publication

After publication:

- Package is immutable in its published state
- All actions remain in the audit trail
- Future updates create new versions (linked to original)
- Client responses are tracked as engagement correspondence
- Package serves as the foundation for audit documentation
