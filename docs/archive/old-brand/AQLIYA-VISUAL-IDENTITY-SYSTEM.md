## 1. BRAND STRATEGY LAYER

### Brand Essence
**"Intelligent infrastructure for enterprise decisions."**

AQLIYA exists at the intersection of human judgment and artificial intelligence — providing structured, proof-driven systems that organizations trust for critical business decisions.

### Brand Personality

| Trait | Description |
|-------|-------------|
| **Intelligent** | Deep analytical capability, not surface-level AI |
| **Trustworthy** | Audit-grade reliability, human-verified outputs |
| **Structured** | Systematic thinking, organized complexity |
| **Premium** | Enterprise-grade quality, not mass-market |
| **Calm** | Confident without being loud |
| **Technical** | Deep capability, accessible presentation |

### Brand Archetype
**The Sage + The Ruler**

- **Sage:** Pursuit of truth, intelligence, evidence-based decisions
- **Ruler:** Order, structure, enterprise authority, governance

### Positioning Language

**We say:**
- "Intelligence that evolves. Impact that lasts."
- "Proof-driven AI for enterprise decisions."
- "Human-reviewed. AI-powered. Enterprise-ready."
- "Where intelligence meets infrastructure."

**We don't say:**
- "Revolutionary AI"
- "Game-changing technology"
- "Next-gen platform"
- "Disrupting the industry"

### Emotional Perception Model

```
Trust ← Intelligence ← Structure ← Clarity ← Confidence
```

The user journey:
1. **First impression:** Calm confidence, professional depth
2. **Engagement:** Structured clarity, systematic thinking
3. **Trust building:** Evidence-based, human-reviewed approach
4. **Loyalty:** Operational reliability, enterprise-grade performance

### Brand Attributes (Weighted)

| Attribute | Weight | Visual Expression |
|-----------|--------|-------------------|
| Trust | 25% | Deep blues, structured layouts, clear hierarchy |
| Intelligence | 20% | Geometric precision, data visualization, systematic icons |
| Enterprise | 20% | Clean grids, professional typography, audit-ready UI |
| Modern | 15% | Subtle gradients, refined spacing, contemporary feel |
| Premium | 10% | Refined details, generous whitespace, quality materials |
| Technical | 10% | System diagrams, structured data, technical metaphors |

### Visual Psychology

- **Deep Blue (#0A0F24):** Authority, trust, depth of intelligence
- **Indigo (#1E3A8A):** Strategic thinking, enterprise stability
- **Cyan (#0EA5E9):** Technology, innovation, forward momentum
- **White/Light:** Clarity, transparency, clean operations

### Tone of Presence

| Channel | Tone |
|---------|------|
| Website | Confident, clear, professional |
| Dashboard | Systematic, precise, operational |
| Reports | Analytical, structured, evidence-based |
| Marketing | Strategic, intelligent, enterprise-focused |
| Social | Thoughtful, insightful, industry-leading |

### Trust Perception Model

```
Visual Trust = (Consistency × Clarity) + (Professionalism × Depth)
```

- **Consistency:** Unified system across all touchpoints
- **Clarity:** No ambiguity, clear hierarchy
- **Professionalism:** Enterprise-grade execution
- **Depth:** Substantive content, not superficial design

## 3. TYPOGRAPHY SYSTEM

### English Typography

**Primary Font: Inter**
- Modern, professional, highly legible
- Excellent for UI and dashboards
- Strong numerical figures for financial data
- Free and open-source

**Secondary Font: Plus Jakarta Sans**
- Geometric, modern
- Great for headings and display
- Complements Inter well

**Primary Arabic Font: IBM Plex Sans Arabic**
- Professional, enterprise-grade
- Excellent readability at all sizes
- Matches Inter's proportions
- Supports all Arabic diacritics

- Good for headings and display

## 4. LOGO USAGE SYSTEM

### Logo Architecture

The AQLIYA logo consists of:
1. **Icon:** Split brain design (organic left + circuit right)
2. **Wordmark:** "AQLIYA" in gradient blue
3. **Tagline:** "MIND THE FUTURE" in smaller text

### Clear Space

```

    [clear space]
    ┌─────────────┐
    │             │
    │   [LOGO]    │
    │             │
    └─────────────┘

```

### Minimum Sizes

| Format | Minimum Size |
|--------|--------------|
| Digital (screen) | 24px height |
| Print | 15mm width |
| Favicon | 16×16px (icon only) |
| App Icon | 1024×1024px |
| Social Avatar | 400×400px |

### Icon Usage

| Context | Format |
|---------|--------|
| Navigation bar | Icon + wordmark |
| Mobile nav | Icon only |
| Favicon | Icon only |
| App icon | Icon only |
| Loading screen | Icon only |
| Watermark | Icon, 10% opacity |

### Color Variations

| Background | Logo Version |
|------------|--------------|
| White/Light | Full color (gradient) |
| Dark (#0A0F24) | White/light version |
| Colored | Monochrome (white or dark) |
| Print (B&W) | Solid black or white |

### Incorrect Usage

❌ Do NOT:
- Stretch or distort the logo
- Change the gradient direction
- Add shadows or effects not in system
- Place on busy backgrounds without clearance
- Rotate the logo
- Change the proportions
- Use unofficial colors
- Add taglines not approved
- Animate the logo excessively

### Dashboard Usage

```
Header: Icon (24px) + "AQLIYA" wordmark
Sidebar: Icon (20px) only
Mobile: Icon (24px) only
Loading: Icon with subtle pulse animation
```

### Responsive Logo System

```
Desktop (>1024px):  Full logo (icon + wordmark + tagline)
Tablet (768-1024px): Icon + wordmark
Mobile (<768px):    Icon only
Favicon:            Icon only (simplified if needed)
```

## 7. MOTION & INTERACTION PRINCIPLES

### Animation Philosophy

**"Intelligent motion — purposeful, not decorative."**

All animations should:
- Serve a functional purpose
- Feel natural and physical
- Be subtle, not distracting
- Maintain performance
- Support accessibility

### Transition Timing

```css
/* Duration Scale */
--duration-instant: 0ms      /* Immediate */
--duration-fast: 100ms       /* Micro-interactions */
--duration-base: 150ms       /* Standard transitions */
--duration-slow: 250ms       /* Complex transitions */
--duration-slower: 350ms     /* Page transitions */
--duration-slowest: 500ms    /* Major state changes */
```

### Easing Functions

```css
/* Easing Scale */
--ease-linear: linear
--ease-in: cubic-bezier(0.4, 0, 1, 1)
--ease-out: cubic-bezier(0, 0, 0.2, 1)
--ease-in-out: cubic-bezier(0.4, 0, 0.2, 1)
--ease-spring: cubic-bezier(0.34, 1.56, 0.64, 1)
```

### Hover Behavior

```css
/* Button hover */
.button {
  transition: all 150ms ease-out;
}
.button:hover {
  transform: translateY(-1px);
  box-shadow: 0 4px 12px rgba(37, 99, 235, 0.3);
}

/* Card hover */
.card {
  transition: all 200ms ease-out;
}
.card:hover {
  transform: translateY(-2px);
  box-shadow: 0 10px 25px rgba(0, 0, 0, 0.1);
}

/* Link hover */
.link {
  transition: color 100ms ease;
}
.link:hover {
  color: #1D4ED8;
}
```

### Loading States

```css
/* Skeleton loading */
.skeleton {
  background: linear-gradient(
    90deg,
    #F1F5F9 25%,
    #E2E8F0 50%,
    #F1F5F9 75%

  0% { background-position: 200% 0; }

/* Spinner */

  animation: spin 1s linear infinite;
}

@keyframes spin {
  from { transform: rotate(0deg); }
  to { transform: rotate(360deg); }
}
```

### AI Interaction Feel

```css
/* AI thinking indicator */
```

### Visual Language for AI

**Core Principle:** AI should feel like an intelligent system, not a gimmick.

### What We Avoid

❌ Robot imagery  
❌ Brain icons (except our logo)  
❌ Glowing neural networks  
❌ Fake futuristic graphics  
❌ "AI" buzzword overload  
❌ Generic chatbot aesthetics

✅ **Intelligence:** Structured data flows, decision trees  
✅ **Orchestration:** Connected systems, workflow visualization  
✅ **Evidence:** Verification badges, audit trails

## 11. DESIGN SYSTEM RULES

### Consistency Rules

1. **Always use the design tokens** — Never hardcode values
2. **Follow the spacing scale** — Use 4px multiples
3. **Maintain typography hierarchy** — Don't skip heading levels
4. **Use approved color combinations** — Check accessibility
5. **Keep logo usage consistent** — Follow clear space rules

### Spacing Rules

```
Internal spacing (within components):  8px, 12px, 16px
External spacing (between components): 16px, 24px, 32px
Section spacing:                        64px, 96px
Page margins:                           24px (mobile), 48px (desktop)
```

### Visual Governance

```
Decision Hierarchy:
1. Brand Strategy → Visual Direction
2. Visual Direction → Design System
3. Design System → Component Library
4. Component Library → Implementation

Change Process:
1. Propose change with rationale
2. Review against brand principles
3. Test across use cases
4. Update documentation
5. Communicate to team
```

### Hierarchy Rules

```
Visual Weight:
Primary:   Bold, large, brand colors
Secondary: Medium, regular size, neutral
Tertiary:  Regular, small, muted

Information Priority:
1. Key metrics/actions (highest contrast)
2. Supporting information (medium contrast)
3. Metadata/context (lowest contrast)
```

### Accessibility Rules

```
1. Minimum contrast ratio: 4.5:1 (AA), 7:1 (AAA)
2. Focus indicators: Always visible
3. Keyboard navigation: Full support
4. Screen readers: Proper ARIA labels
5. Color independence: Don't rely on color alone
6. Motion: Respect prefers-reduced-motion
7. Text: Minimum 16px body, scalable
```

### Enterprise Scalability Principles

```
1. Token-based design — All values from tokens
2. Component composition — Build from primitives
3. Theme support — Light/dark mode ready
4. Responsive by default — Mobile-first approach
5. Internationalization — RTL support, translations
6. Performance — Optimized assets, lazy loading
7. Documentation — Living design system docs
```
