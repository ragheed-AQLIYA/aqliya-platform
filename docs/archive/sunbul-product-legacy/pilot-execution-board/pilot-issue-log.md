## Severity Definitions

| Severity | Definition | Response Time |
|---|---|---|
| **Blocker** | Pilot cannot continue. User cannot complete a core workflow step. | Immediate — within 4 hours |
| **High** | Major workflow disruption but workaround exists. User can continue with manual steps. | Within 24 hours |
| **Medium** | Non-critical issue. Feature works but has inconvenience. | Within 1 week |
| **Low** | Cosmetic, label, or nice-to-have improvement. No impact on workflow completion. | Within 2 weeks or backlog |

## Status Definitions

| Status | Meaning |
|---|---|
| **Open** | Issue reported, not yet addressed |
| **In Progress** | Being investigated or fixed |
| **Resolved** | Fix applied and verified |
| **Won't Fix** | Issue accepted as known limitation |
| **Deferred** | Will be addressed in a later phase |
