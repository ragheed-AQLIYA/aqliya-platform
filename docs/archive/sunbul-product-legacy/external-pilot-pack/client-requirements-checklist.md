## Required Reviewer / Approver

- [ ] Reviewer confirmed and available during pilot
- [ ] Reviewer has authority to approve cases (or at least review them)
- [ ] Reviewer can dedicate 1–2 hours per week

## Required Test Scenarios

The client should be able to test:

- [ ] Create a case with title and description
- [ ] Upload at least one document
- [ ] Submit case for review
- [ ] Review case and return with notes
- [ ] Revise and resubmit
- [ ] Approve case
- [ ] Export approved case as PDF
- [ ] View audit trail
- [ ] Verify case details in exported PDF

## Access Requirements

- [ ] Internet connection (cloud deployment)
- [ ] Modern browser (Chrome, Edge, or Firefox — latest version)
- [ ] No VPN or firewall that blocks the platform URL
- [ ] No browser extension that blocks form submission

## Data Sensitivity Confirmation

- [ ] Pilot data does not include highly sensitive personal information (PID)
- [ ] Pilot data does not include bank account numbers or credit card data
- [ ] Pilot data does not include government-classified information
- [ ] Client confirms they accept the current security model (database encryption, no at-rest file encryption)
