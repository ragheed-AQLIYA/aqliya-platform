# Sunbul — Demo Agenda

**Duration:** 30–45 minutes
**Format:** Screen share (Google Meet / Zoom / Teams)
**Audience:** Pilot client team (operator, reviewer, decision maker)

---

## Agenda

| Time | Topic | Lead | Format |
|---|---|---|---|
| 0:00–0:03 | Opening & introductions | Sunbul lead | Verbal |
| 0:03–0:07 | Problem framing | Sunbul lead | Screen + story |
| 0:07–0:15 | Workflow walkthrough | Sunbul lead | Live demo |
| 0:15–0:25 | Client-specific case creation | Client operator | Hands-on |
| 0:25–0:30 | Document upload | Client operator | Hands-on |
| 0:30–0:33 | Review and return | Client reviewer | Hands-on |
| 0:33–0:35 | Revise and resubmit | Client operator | Hands-on |
| 0:35–0:38 | Approval | Client reviewer | Hands-on |
| 0:38–0:40 | PDF export | Client operator | Hands-on |
| 0:40–0:42 | Audit trail review | Sunbul lead | Screen |
| 0:42–0:45 | Questions & next steps | All | Discussion |

## Detailed Script

### 0:00–0:03 — Opening

Introductions. Explain the demo structure. Confirm recording if needed.

### 0:03–0:07 — Problem Framing

سؤال: كيف تديرون حالياً طلبات المراجعة والموافقة على المستندات؟

Walk through the common problems:
- ملفات مبعثرة على الإيميل
- لا يوجد مسار تدقيق واضح
- صعوبة تتبع من اطلع على الملف ومن اعتمده
- لا يوجد تقرير موحد يربط القضية بالمستندات والموافقات

Introduce Sunbul as the solution.

### 0:07–0:15 — Workflow Walkthrough (Screen Share)

1. Open `/sunbul` — show the workspace landing page.
2. Select the pilot client from the dropdown.
3. Point out:
   - Stat cards (total, draft, under review, approved)
   - Record list
   - Review queue
   - Create case button
4. Open an existing draft case (or create one).
5. Walk through the case detail page:
   - Status badge
   - Description
   - Workflow actions
   - Document panel
   - Review panel
   - Audit trail
   - Export section

### 0:15–0:25 — Client Creates a Case (Hands-on)

Ask the client operator to:
1. Click "إنشاء قضية"
2. Enter a title from their real work
3. Enter a brief description
4. Submit and verify it appears in the list
5. Open the case and verify the status shows "مسودة"

### 0:25–0:30 — Document Upload (Hands-on)

Ask the client operator to:
1. Click "رفع مستند"
2. Select a PDF or image from their computer
3. Wait for upload to complete
4. Verify the document appears in the document list

### 0:30–0:33 — Submit and Review (Hands-on)

**Operator:**
1. Click "إرسال للمراجعة"
2. Verify status changes to "تحت المراجعة"

**Reviewer:**
1. From the dashboard, see the case in the review queue
2. Open the case
3. Click "إرجاع", add a comment: "يرجى إضافة تفاصيل إضافية"
4. Confirm

### 0:33–0:35 — Revise and Resubmit (Hands-on)

**Operator:**
1. Open the returned case
2. See the return notes
3. Update the description
4. Click "إرسال للمراجعة" again

### 0:35–0:38 — Approval (Hands-on)

**Reviewer:**
1. Open the resubmitted case from the review queue
2. Click "اعتماد"
3. Verify status changes to "معتمد"

### 0:38–0:40 — PDF Export (Hands-on)

**Operator:**
1. Open the approved case
2. In the Export section, click "تحميل التقرير (PDF)"
3. Open the downloaded PDF
4. Verify: title, status, description, documents, reviews, audit trail, disclaimer

### 0:40–0:42 — Audit Trail (Screen Share)

1. Scroll to the audit trail section
2. Show the timeline of events with timestamps
3. Point out: who did what, when

### 0:42–0:45 — Questions & Next Steps

- Ask the client what they think
- Ask if this workflow matches their real process
- Ask what's missing
- Set the pilot start date
- Confirm who does what
- Share the operator manual
