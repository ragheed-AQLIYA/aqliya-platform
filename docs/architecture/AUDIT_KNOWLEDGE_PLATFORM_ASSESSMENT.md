# Audit Knowledge Platform Assessment

**Date:** 2026-06-15  
**Evidence:** Phase 3A–3D validation artifacts, Shalfa pilot benchmarks, governance policy in repo (uncommitted)  
**Principle:** AI assists. Humans decide. Evidence governs.

---

## Positioning Question

Can AuditOS be positioned as an **Audit Knowledge Platform** instead of an **AI Classification Platform**?

**Answer: YES — with explicit boundaries.**  
Git + validation evidence supports repositioning for **same-ERP reuse**; does **not** support general AI classifier claims.

---

## Evidence Summary

| Layer | Result (Shalfa) | Implication |
|-------|-----------------|-------------|
| Rules only | ~11% | Not the moat |
| Local AI | ~20% | Assist tier only |
| Hybrid / ERP hints | ~48% | Useful, not durable moat |
| Firm Memory (same ERP) | **100%** (578/578) | **Core moat** |
| TRUSTED auto-suggest (Year 1 backfill) | **0%** | Governance working as designed |

---

## Strengths

1. **Firm Memory** — accumulated confirmations become reusable audit knowledge
2. **Governance** — TRUSTED requires hits + reviewers + freshness; not confidence theater
3. **Explainability (Phase 4)** — Trust + Evidence UI/API, not confidence-first UX
4. **Factory stack** — FS engine, presentation policies, IFRS/SOCPA rules (institutional workflow)
5. **Human-in-the-loop** — confirmations write memory; AI does not auto-certify

---

## Weaknesses

1. **Not cross-ERP generalization** — hold-out stratified ~36%; do not sell as universal classifier
2. **Reuse rate baseline 0%** on history source field — KPI immature until Client 2
3. **Recovery risk** — until git commit/push, knowledge asset is not institutionally secured
4. **Local AI** — deferred correctly; no evidence to lead with AI positioning
5. **Production gap** — migrations + code not on `origin/main` yet

---

## Moat Analysis

```text
Durable moat = Audit Factory workflow
              + Accumulated Firm Memory (per org / ERP chart)
              + Governance trust lifecycle
              + Evidence-linked mappings

NOT moat = LLM classification accuracy
         = RAG / embeddings
         = Cross-client zero-shot generalization
```

**Commercial framing:**

> AuditOS under AQLIYA is an **Audit Knowledge Platform** that captures how your firm mapped, reviewed, and confirmed accounts — and reuses that knowledge on the next engagement with the same ERP.
