# 🟢 AuditOS — قائمة طلب أدلة المراجعة

## آلية العمل

النظام يمكّن فريق المراجعة من طلب الأدلة وتتبّعها داخل المنصة. دورة حياة الدليل:

```
مطلوب (Missing) ← مرفوع (Uploaded) ← مقيّد (Linked) ← مراجع (Reviewed) ← مقبول (Accepted)
                                                                       ← مرفوض (Rejected)
```

---

## ١. تصنيف الأدلة حسب الغرض

| الغرض | نوع الدليل | أمثلة |
|-------|-----------|-------|
| **وجود** (Existence) | إثبات وجود الأصل أو الالتزام | كشف بنكي، جرد مخزون، سند ملكية |
| **دقة** (Accuracy) | التحقق من صحة المبلغ | فاتورة شراء، كشف حساب مورد |
| **تقييم** (Valuation) | تقييم الأرصدة | تقرير تقييم عقار، دراسة انخفاض قيمة |
| **اكتمال** (Completeness) | التحقق من عدم وجود بنود محذوفة | تحليل مصروفات، تسوية مبيعات |
| **إفصاح** (Presentation) | التأكد من كفاية الإفصاح | محضر مجلس إدارة، عقد إيجار، خطاب ضمان |
| **حقوق والتزامات** (Rights & Obligations) | تأكيد الملكية أو الالتزام | سند ملكية، عقد قرض، إقرار ضريبي |

## ٣. تعليمات لفريق المراجعة

| # | تعليمات |
|---|---------|
| ١ | لكل دليل مطلوب: حدد اسم الدليل ونوعه والغرض منه في النظام. |
| ٢ | النظام يتيح رفع الدليل مباشرة عند استلامه. |
| ٣ | بعد الرفع: ينشئ النظام تجزئة SHA-256 لحفظ سلامة الملف. |
| ٤ | ربط الدليل بالملاحظة ذات العلاقة من داخل شاشة الملاحظة أو الدليل. |
| ٥ | كل تغيير في حالة الدليل يُسجّل في سجل التدقيق. |
| ٦ | استخدم خاصية "الأدلة الناقصة" في لوحة التحكم لمتابعة ما لم يُستلم. |

## ٥. تدقيق مقبولية الدليل

عند مراجعة الدليل المرفوع، يتحقق المراجع من:

- [ ] الدليل مطابق للصنف المطلوب (ليس ملفًا آخر).
- [ ] التاريخ يقع ضمن فترة المراجعة.
- [ ] الجهة المصدرة موثوقة (داخلية أم خارجية).
- [ ] الملف واضح ومقروء.
- [ ] كمية الأدلة كافية للحكم المهني.
