# 🟢 AuditOS — متطلبات تقديم ميزان المراجعة

## ١. الصيغة المطلوبة

| الخيار | التفضيل | ملاحظات |
|--------|---------|---------|
| **XLSX** | ⭐ مفضّل | أفضل دعم للغة العربية وتنسيق الخلايا |
| CSV | ✅ مقبول | UTF-8 encoding مطلوب |

---

## ٢. هيكل الملف (الأعمدة الدنيا المطلوبة)

| العمود | الوصف | مثال | إلزامي؟ |
|--------|-------|------|---------|
| `Account Code` | رقم / كود الحساب | `1010` أو `1-01-001` | ✅ |
| `Account Name` | اسم الحساب (عربي أو إنجليزي) | `نقدية بالصندوق` | ✅ |
| `Debit` | إجمالي الحركة المدينة للفترة | `150000.00` | ✅ |
| `Credit` | إجمالي الحركة الدائنة للفترة | `35000.00` | ✅ |
| `Balance` | الرصيد في نهاية الفترة | `115000.00` | ✅ |
| `Account Type` | تصنيف الحساب | `أصل` / `التزام` / `إيراد` / `مصروف` | ✅ |
| `Opening Balance` | الرصيد الافتتاحي | `100000.00` | اختياري |

> **مهم:** يجب أن تكون الأرصدة المدينة بالموجب (`+`) والأرصدة الدائنة بالسالب (`-`) أو العكس بشكل متسق على جميع الحسابات.

---

## ٣. تعليمات تنسيق الملف

### إذا كان الملف XLSX:

- استخدام لسان تبويب (Sheet) واحد فقط.
- الصف الأول = رؤوس الأعمدة (Headers).
- لا تستخدم صفوفًا مدمجة (Merged Cells).
- لا تستخدم مرشحات (Filters) أو جداول محورية (Pivot Tables).
- الأرقام بصيغة رقمية بحتة (بدون فواصل الآلاف النصية).
- تأكد من عدم وجود معادلات (Formulas) — يفضل نسخ القيم فقط (Paste as Values).

### إذا كان الملف CSV:

- الترميز (Encoding): **UTF-8** (وليس UTF-8 BOM).
- الفاصل (Delimiter): فاصلة `,` (Comma).
- النصوص التي تحتوي فواصل: تُحاط بعلامات تنصيص `"..."`.
- لا تحتوي على أسطر فارغة في المنتصف.

## ٥. فحص الجودة قبل التقديم

| # | الفحص | ✓/✗ |
|---|-------|-----|
| ١ | مجموع الأرصدة المدينة = مجموع الأرصدة الدائنة (أو الانحراف أقل من ١٪) | ☐ |
| ٢ | جميع الحسابات لها أكواد فريدة | ☐ |
| ٣ | جميع الحسابات لها تصنيف (Account Type) | ☐ |
| ٤ | لا توجد صفوف فارغة في المنتصف | ☐ |
| ٥ | الأسماء واضحة وليست مقتطعة | ☐ |
| ٦ | لا توجد صفوف إجماليات يدوية (سيحسبها النظام) | ☐ |
| ٧ | الأرقام لا تحتوي على رموز عملة أو نصوص | ☐ |

## ٦. الأخطاء الشائعة (تجنّبها)

| الخطأ | التصحيح |
|-------|---------|
| دمج صف العناوين مع صفوف البيانات | فصل الصفوف |
| وجود صف "إجمالي" في منتصف الملف | احذف جميع صفوف الإجماليات — النظام يحسبها |
| تنسيق الأرقام كنص | حوّل الأعمدة الرقمية إلى Number |
| استخدام علامة `-` بدل الرقم السالب `-` | استخدم نفس الرمز لكل الحسابات |
| وجود حسابات بدون تصنيف | أضف تصنيفًا لكل حساب ولو مؤقتًا |

---

## ٧. بعد رفع الملف

سيتحقق النظام من:
- وجود جميع الأعمدة الإلزامية.
- عدد الحسابات وتصنيفاتها.
- الانحراف بين مجموع الدائن والمدين (Trial Balance Variance).
- حالة الثقة (Trusted / Conditional / Blocked) بناءً على جودة البيانات.

سيتم إعلامكم فورًا بأي أخطاء للتصحيح قبل المتابعة.
