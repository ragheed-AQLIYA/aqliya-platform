## ٢. مسؤوليات AQLIYA

| # | المسؤولية | تفصيل |
|---|----------|--------|
| ١ | **تجهيز البيئة التقنية** | تثبيت المنصة وتهيئتها في بيئة البايلوت |
| ٢ | **إنشاء الحسابات وإدارة الصلاحيات** | إعداد مستخدمي العميل بالأدوار الصحيحة |
| ٣ | **تقديم التدريب الكامل** | ٣ جلسات تدريبية تغطي سير العمل كاملًا |
| ٤ | **الدعم الفني خلال البايلوت** | الرد على الاستفسارات خلال ٢٤ ساعة (أيام العمل) |
| ٥ | **جلسات متابعة دورية** | أسبوعيًا حسب الجدول المختار |
| ٦ | **توثيق الملاحظات والتغذية الراجعة** | تسجيل جميع ملاحظات العميل في سجل Pilot Feedback |
| ٧ | **معالجة الأعطال التقنية** | أي خطأ يمنع إكمال سير العمل |
| ٨ | **تقييم جودة البيانات المدخلة** | مراجعة ميزان المراجعة بعد الرفع الأولي |
| ٩ | **تقديم تقرير تقييم نهائي** | تقرير Go/No-Go مع بطاقة النتيجة |
| ١٠ | **اقتراح خطة تحول للإنتاج** | خطة ما بعد البايلوت |

## ٣. مسؤوليات العميل (مكتب المراجعة)

| # | المسؤولية | تفصيل |
|---|----------|--------|
| ١ | **ترشيح منسق داخلي** | نقطة اتصال واحدة مسؤولة عن التنسيق |
| ٢ | **تجهيز البيانات المطلوبة** | ميزان مراجعة، أدلة، بيانات المنشأة |
| ٣ | **تخصيص أعضاء الفريق** | ٢–٤ مستخدمين: مُعدّ + مراجع + شريك |
| ٤ | **حضور جلسات التدريب** | جميع أعضاء الفريق المخصصين |
| ٥ | **استخدام المنصة على بيانات حقيقية** | تنفيذ سير العمل فعليًا |
| ٦ | **توثيق الملاحظات** | إبداء الملاحظات على سير العمل وجودة المخرجات |
| ٧ | **تعبئة استبيان التقييم** | في نهاية فترة البايلوت |
| ٨ | **المشاركة في اجتماع الإغلاق** | مراجعة النتائج واتخاذ قرار المرحلة التالية |
| ٩ | **الحفاظ على سرية بيانات الدخول** | عدم مشاركة كلمات المرور |
| ١٠ | **استخدام ملفات موثوقة المصدر** | نظرًا لعدم وجود فحص فيروسات في البايلوت |

## ٤. مصفوفة RACI مختصرة

| النشاط | AQLIYA | العميل |
|--------|--------|--------|
| توفير المنصة وتشغيلها | **A** (مسؤول) | I (مُبلغ) |
| تجهيز البيانات والملفات | I (مُبلغ) | **A** (مسؤول) |
| تدريب المستخدمين | **A** (مسؤول) | R (منفّذ بحضور) |
| استخدام المنصة | C (مستشار) | **A** (مسؤول) |
| تصنيف الحسابات | C (مستشار) | **A** (مسؤول) |
| إدارة الأدلة والملاحظات | I (مُبلغ) | **A** (مسؤول) |
| المراجعة والاعتماد | I (مُبلغ) | **A** (مسؤول) |
| حل المشكلات التقنية | **A** (مسؤول) | I (مُبلغ) |
| تقييم التجربة | R (منفّذ) | **A** (مسؤول) |
| قرار Go/No-Go | C (مستشار) | **A** (مسؤول) |

> **A** = مسؤول (Accountable) | **R** = منفّذ (Responsible) | **C** = مستشار (Consulted) | **I** = مُبلغ (Informed)

## ٥. الأدوار داخل النظام

| الدور | من يقوم به؟ | المهام الرئيسية |
|-------|------------|----------------|
| **Admin** | AQLIYA (تقني) | إنشاء حسابات، إدارة النظام، ضبط الإعدادات |
| **Operator** | مُعدّ من فريق العميل | رفع ميزان المراجعة، تصنيف الحسابات، رفع الأدلة، إنشاء الملاحظات |
| **Reviewer** | مراجع / مدير تدقيق من العميل | مراجعة العناصر، كتابة التعليقات، حلّها |
| **Partner** | الشريك المعتمد من العميل | اعتماد الارتباط أو رفضه مع إبداء الأسباب |
| **Viewer** | أي عضو يحتاج للاطلاع فقط | تصفّح جميع العناصر دون تعديل |
