# Disclosure Auto Engine — AuditOS 2.0 Phase 8

**Status:** Implemented (feature-flagged, default off)  
**Flag:** `audit.disclosure-auto` / `FF_AUDIT_DISCLOSURE_AUTO=true`

## Purpose

Materialize IFRS and SOCPA **disclosure triggers** into governed draft `AuditDisclosureNote` records:

- Collect triggers from rules engines (when their flags are on)
- Dedupe by note type + title + source + rule
- Create or update **draft** notes with `aiDrafted: true`
- Embed `RULE_CITATION|…` markers in `missingInformation`
- **Never overwrite** notes in `reviewed` or `approved` status
- Optionally chain **Audit Intelligence** enrichment after run

## Pipeline Position

```
FS Rebuild → IFRS Rules → SOCPA Rules → Disclosure Auto → Intelligence (optional)
```

| Surface | Hook |
| ------- | ---- |
| Post-SOCPA | `maybeAutoGenerateDisclosureNotes` in `rebuildFinancialStatementsForEngagement` |
| Manual | `DisclosureAutoPanel` on notes tab |
| Audit log | `disclosure.auto_completed` |

## Module

| File | Role |
| ---- | ---- |
| `disclosure-engine-types.ts` | Unified trigger + run result types |
| `disclosure-engine.ts` | Pure unify, dedupe, draft builder |
| `disclosure-auto.ts` | Persist, flag, hooks (import direct path — not client barrel) |
| `disclosure-types.ts` | Rule citation markers (shared) |

## Dependencies

Disclosure auto produces notes only when **rule triggers exist**:

| Flag | Required for |
| ---- | ------------ |
| `FF_AUDIT_IFRS_RULES` | IFRS disclosure triggers |
| `FF_AUDIT_SOCPA_RULES` | SOCPA zakat/tax triggers (Saudi jurisdiction) |

Recommended factory stack:

```env
FF_AUDIT_FS_V2=true
FF_AUDIT_IFRS_RULES=true
FF_AUDIT_SOCPA_RULES=true
FF_AUDIT_DISCLOSURE_AUTO=true
FF_AUDIT_INTELLIGENCE=true
```

## Governance

- All generated content is **assistive draft** — human review required
- `HUMAN_REVIEW_REQUIRED` marker on every auto note
- Reviewed/approved notes are skipped (no silent overwrite)
- Server actions require engagement access + RBAC
