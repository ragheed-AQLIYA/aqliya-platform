# Target Users

## Audit Partner

| Aspect | Detail |
|--------|--------|
| Title | Partner / Managing Partner / Owner |
| Goals | Firm scalability, quality control, risk reduction, regulatory confidence |
| Pains | Review bottlenecks, inconsistent reviewer quality, untraceable decisions, liability exposure, difficulty scaling without adding partners |
| Primary workflows | Engagement overview, exception escalation, report approval, firm-wide quality metrics |
| System interaction | Dashboard-level, exception-based, governance configuration |
| Success metric | More engagements at same quality with less partner time |

## Audit Manager

| Aspect | Detail |
|--------|--------|
| Title | Audit Manager / Senior Manager |
| Goals | Engagement throughput, staff productivity, evidence completeness, review quality |
| Pains | Chasing evidence, manual review assignments, inconsistent finding quality, status reporting, rework from incomplete evidence |
| Primary workflows | Engagement setup, staff assignment, review queue management, finding approval |
| System interaction | Workflow-heavy, manages exceptions and escalations |
| Success metric | Reduced review cycle time, complete evidence coverage |

## Reviewer / Senior Auditor

| Aspect | Detail |
|--------|--------|
| Title | Senior Auditor / Reviewing Accountant |
| Goals | Efficient evidence review, clear findings, defensible recommendations, minimal administrative overhead |
| Pains | Switching between systems for evidence, findings, and review; untraceable review comments; repetitive manual assessment; finding format inconsistency |
| Primary workflows | Evidence review, signal assessment, finding creation, recommendation review |
| System interaction | Primary daily user — spends 6-8 hours in the system |
| Success metric | Faster finding creation, fewer follow-up review cycles |

## Operator / Preparer

| Aspect | Detail |
|--------|--------|
| Title | Junior Accountant / Audit Assistant / Preparer |
| Goals | Efficient data preparation, clear evidence collection, structured finding drafting |
| Pains | Manual data extraction, evidence formatting uncertainty, unclear finding format expectations |
| Primary workflows | Trial balance upload, evidence upload, evidence linking, initial finding drafting |
| System interaction | Structured data entry, guided workflows |
| Success metric | Less rework, clearer guidance on what to prepare |

## Client-Side Finance Contact

| Aspect | Detail |
|--------|--------|
| Title | CFO / Finance Manager / Accountant |
| Goals | Understand audit findings, provide supporting data, receive clear requests |
| Pains | Unknown evidence requirements, status uncertainty, delayed feedback, unclear adjustment rationale |
| Primary workflows | Data submission, evidence upload, published recommendation review |
| System interaction | Limited — portal-style access for data upload and output review |
| Success metric | Clear evidence requests, faster response cycle |

## Admin

| Aspect | Detail |
|--------|--------|
| Title | System Administrator / IT |
| Goals | Tenant setup, user management, system configuration, deployment |
| Pains | Manual user provisioning, inconsistent permission setup, deployment complexity |
| Primary workflows | Organization setup, user creation, role assignment |
| System interaction | Configuration-focused, infrequent |
| Success metric | Self-service user management, auditable access control |
