# Sombol — AQLIYA Implementation Scope

**Version:** 1.0
**Status:** Phased implementation plan
**Classification:** Cloud-first, Private-ready architecture

---

## Phase 1: Cloud Pilot (0–3 months)

### AQLIYA Responsibilities

| المهمة | الموعد |
|---|---|
| إعداد بيئة Cloud لـ Sombol | الأسبوع 1 |
| إنشاء PlatformOrganization | الأسبوع 1 |
| إنشاء ClientWorkspace للعملاء المختارين | الأسبوع 1 |
| تهيئة AuditOS engagements للبيانات التجريبية | الأسبوع 1–2 |
| تدريب المستخدمين على AuditOS | الأسبوع 2–3 |
| تدريب المستخدمين على Office AI Assistant | الأسبوع 3–4 |
| الدعم الفني المستمر | طوال Pilot |

### Sombol Responsibilities

| المهمة | الموعد |
|---|---|
| توفير قائمة العملاء المختارين للتجربة | قبل الأسبوع 1 |
| توفير فريق التدقيق المشارك | الأسبوع 1 |
| توفير بيانات اختبارية (ميزان مراجعة، مستندات) | الأسبوع 1–2 |
| المشاركة في جلسات التدريب | الأسبوع 2–4 |
| تقديم ملاحظات أسبوعية | كل أسبوع |
| تقييم المرحلة التجريبية نهاية الشهر الثالث | الأسبوع 12 |

### Deliverables

| التسليم | التاريخ |
|---|---|
| بيئة Cloud عاملة مع AuditOS | الأسبوع 1 |
| مساحات عمل العملاء جاهزة | الأسبوع 1 |
| مهمة Office AI Assistant عاملة | الأسبوع 2 |
| سجل تدقيق موحد يعرض جميع الأحداث | الأسبوع 2 |
| تقرير تقييم المرحلة التجريبية | الأسبوع 12 |

### Technical Assumptions

- Sombol توفر اتصال إنترنت مستقر
- المستخدمون يستخدمون متصفحات حديثة (Chrome, Edge, Firefox)
- الملفات المرفوعة لا تتجاوز 10 ميجابايت لكل ملف
- لا حاجة لتكامل مع Active Directory أو SSO في Pilot
- لا حاجة لاستضافة داخلية في Pilot

## Phase 2: Cloud Expansion (3–6 months)

### When to Start

- نجاح المرحلة التجريبية
- اتخاذ قرار التوسع من قبل إدارة Sombol
- توقيع اتفاقية تجارية

### Scope

| البند | الوصف |
|---|---|
| **جميع العملاء** | نقل جميع عملاء Sombol إلى AQLIYA Cloud |
| **جميع المستخدمين** | توسيع الحسابات لجميع الموظفين (حتى 50) |
| **Cloud AI** | ربط مزود ذكاء اصطناعي سحابي (OpenAI/Claude) — إذا تم تطويره |
| **تحليل الملفات** | تحليل محتوى PDF/Word/Excel — إذا تم تطويره |
| **Office AI Assistant كامل** | جميع أنواع المهام مع تحليل حقيقي للمحتوى |
| **تقارير مخصصة** | قوالب تقارير مخصصة لـ Sombol |
| **دعم فني موسع** | قناة دعم مخصصة + ضمان تشغيل 99.5% |

### AQLIYA Responsibilities

- تفعيل الحسابات لجميع المستخدمين
- نقل بيانات العملاء من Pilot إلى الإنتاج
- تكوين قوالب التقارير
- تدريب إضافي
- دعم فني

### Sombol Responsibilities

- الموافقة على اتفاقية الخدمة
- توفير بيانات جميع العملاء
- تعيين مسؤول منصة داخلي
- التنسيق مع AQLIYA لجلسات التدريب

---

## Phase 3: Private / Internal Deployment (6–12 months)

### Trigger

- اكتمال مرحلة Cloud Expansion
- طلب صريح من Sombol للنشر الداخلي
- إتمام المتطلبات التقنية (أنظر private-deployment-next-phase.md)

### Scope

- نشر AQLIYA داخل بيئة Sombol (خوادم محلية)
- قاعدة بيانات محلية
- تخزين ملفات محلي
- ذكاء اصطناعي محلي (اختياري — عند التطوير)
- لا حاجة لاتصال إنترنت خارجي للتشغيل الأساسي

### Limitations (Must Acknowledge)

| العنصر | الحالة |
|---|---|
| On-Prem كحزمة إنتاج | غير جاهزة — تحت التطوير |
| Local AI | غير مطبّق — في خارطة الطريق |
| Air-Gapped | غير مطبّق — يتطلب On-Prem أولاً |
| الترقية التلقائية | يدوية — لا توجد آلية تحديث تلقائي |
