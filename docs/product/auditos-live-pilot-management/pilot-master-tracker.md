# AQLIYA AuditOS Pilot Master Tracker

Source of truth:

1. `docs/product/auditos-product-packaging.md`
2. `docs/product/auditos-outbound-kit/`
3. `docs/product/auditos-sales-ops/`

## Purpose

هذا المستند هو القالب الرئيسي لتتبع أول 3-5 pilot customers بشكل أسبوعي وعملي.

## Recommended Fields

| Field | Description |
|---|---|
| Pilot ID | معرف داخلي مختصر |
| Account Name | اسم العميل |
| Segment | Audit firm / accounting office / CFO team / SME finance |
| Buyer | اسم صاحب القرار |
| Internal Owner | المسؤول الداخلي عن الحساب |
| Start Date | تاريخ بداية الـ pilot |
| Planned End Date | تاريخ النهاية المخطط |
| Current Status | onboarding / active / blocked / review / closeout |
| Primary Use Case | use case الأساسية |
| Main Pain | المشكلة الرئيسية |
| Success Criteria | معايير النجاح المختصرة |
| Current Risk Level | low / medium / high |
| Next Action | الخطوة التالية |
| Next Action Owner | من المسؤول عنها |
| Next Action Date | تاريخ التنفيذ |
| Post-Pilot Decision | proposal / extend / pause / stop |

## Tracking Rule

كل pilot يجب أن تحتوي دائمًا على:

1. owner واضح
2. تاريخ بداية ونهاية
3. use case أساسية
4. status حالية
5. risk note
6. next action

## Suggested Status Meanings

1. `onboarding` — تم الاتفاق والبدء بالتجهيز
2. `active` — الـ pilot تعمل فعليًا
3. `blocked` — يوجد عائق يوقف التقدم
4. `review` — يتم تقييم النتائج أو evidence
5. `closeout` — مرحلة القرار النهائي بعد الـ pilot

## Weekly Update Format

لكل حساب حدّث:

1. ما الذي حدث هذا الأسبوع؟
2. ما أهم evidence تم جمعها؟
3. هل يوجد blocker؟
4. ما next action؟
5. هل ما زال الـ pilot على المسار الصحيح؟
