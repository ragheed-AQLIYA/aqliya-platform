## Qualification Criteria

### Must-Have

- [ ] Annual revenue or audit portfolio of 5–50 engagements
- [ ] Currently using manual/spreadsheet-based audit workflow
- [ ] Concerned about governance, evidence, or review trails
- [ ] Arabic-speaking finance team (primary language)
- [ ] Willing to dedicate 1 hour for a controlled pilot session
- [ ] Has sample anonymized trial balance data (or accepts demo data)
- [ ] Decision-maker present during session (partner, CFO, manager)

### Nice-to-Have

- [ ] Previously expressed interest in audit automation
- [ ] Familiar with IFRS for SMEs or Saudi GAAP
- [ ] Has experienced audit process pain (lost evidence, review bottlenecks)
- [ ] Open to participating in a case study after pilot
- [ ] Has a network for referrals (other firms, peers)

### Exclusion Criteria

- [ ] Currently using a competing audit automation platform
- [ ] Requires production SLA or compliance certification immediately
- [ ] Not willing to use demo/synthetic data
- [ ] Expects AI to make final audit decisions
- [ ] Cannot commit to a 1-hour session

## Approach Strategy

| Stage                | Action                                                 | Owner |
| -------------------- | ------------------------------------------------------ | ----- |
| 1. Research          | Identify specific firms matching profiles              |       |
| 2. Warm intro        | LinkedIn, referral, existing network                   |       |
| 3. Value proposition | "Governed audit intelligence with evidence and review" |       |
| 4. Session invite    | "45-minute controlled demo using your data or ours"    |       |
| 5. Confirm           | Share constraints doc (08), confirm understanding      |       |
| 6. Schedule          | Per session management playbook (10)                   |       |

## Triage

| Signal                        | Action                                                              |
| ----------------------------- | ------------------------------------------------------------------- |
| "When can we buy?"            | Explain controlled pilot, set expectations, do NOT promise timeline |
| "Can it replace our auditor?" | No — AuditOS assists the auditor, does not replace                  |
| "Is this AI-approved?"        | Explain: AI assists, human decides. Not autonomous.                 |
| "How is data protected?"      | Refer to constraints doc (08), explain tenant isolation             |
| "I want 20 more features"     | Defer to scope creep policy (12), capture in feedback               |
