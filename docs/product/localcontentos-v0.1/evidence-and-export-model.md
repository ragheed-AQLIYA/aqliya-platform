# LocalContentOS v0.1 — Evidence & Export Model

## Evidence Model

### Evidence Types

| Type           | Description                                    | Required For               |
| -------------- | ---------------------------------------------- | -------------------------- |
| `certificate`  | Local content certificate from authorized body | Classification > 50% local |
| `contract`     | Signed contract or agreement                   | Spend record               |
| `attestation`  | Self-declaration or company attestation        | Classification < 50% local |
| `invoice`      | Invoice or payment record                      | Spend record verification  |
| `registration` | Commercial registration or license             | Supplier registration      |
| `other`        | Supporting document                            | As needed                  |

### Evidence Statuses

| Status     | Meaning                                       |
| ---------- | --------------------------------------------- |
| `uploaded` | File stored, not yet linked to records        |
| `linked`   | File linked to supplier/spend/finding         |
| `reviewed` | Human reviewed and accepted                   |
| `verified` | Cross-checked against another source          |
| `rejected` | Reviewed and rejected (insufficient, invalid) |
| `missing`  | Required but not yet provided                 |

### Evidence Requirements Per Domain

| Domain                  | Required Evidence                  | Optional Evidence         |
| ----------------------- | ---------------------------------- | ------------------------- |
| Supplier registration   | CR number, registration document   | Additional certifications |
| High local claim (>50%) | Local content certificate          | Workforce data            |
| Low local claim (<=50%) | Analyst note or contract terms     | Self-declaration          |
| Spend record            | Contract reference, PO, or invoice | Delivery receipt          |
| Finding                 | Evidence of the gap or risk        | Resolution evidence       |

### File Handling

- Storage: reuse `src/lib/platform/storage/` (same as AuditOS, Sunbul)
- Max file size: 10MB per file
- Allowed types: pdf, xlsx, docx, jpg, png, csv, txt
- Checksums: SHA-256 on upload
- Download: permissioned via project access guard
- UI: File upload form wired at `/local-content/projects/[projectId]/evidence` via `EvidenceFileUploadForm` component (L5 patch — 2026-05-21)
- Metadata-only form also available via `EvidenceForm` for reference/linked evidence

## Export Model

### Export Outputs

| Report                           | Format     | Content                                                                        |
| -------------------------------- | ---------- | ------------------------------------------------------------------------------ |
| Local Content Assessment Summary | PDF        | Cover, score, breakdown, status, reviewer, approver, disclaimer                |
| Supplier Locality Register       | XLSX       | Supplier name, CR, classification, percentage, evidence status                 |
| Spend Classification Report      | XLSX       | Spend by supplier/category, local/non-local split, totals                      |
| Gap & Risk Register              | PDF + XLSX | Findings list, severity, status, linked evidence                               |
| Evidence Index                   | XLSX       | All evidence items, status, linked records, review state                       |
| Final Export Package             | PDF        | Combined: summary + register + findings + evidence index + review/approval log |

### Export Metadata

Every export must include a cover/disclaimer page:

```
═══════════════════════════════════════════
LocalContentOS — Local Content Assessment
═══════════════════════════════════════════

Project: [name]
Organization: [org name]
Reporting Period: [period]
Generated: [timestamp]
Generated by: [user name]

Review Status: [reviewed / pending]
Reviewed by: [reviewer name]
Review Date: [date]

Approval Status: [approved / rejected / pending]
Approved by: [approver name]
Approval Date: [date]

⚠ DISCLAIMER
This report is generated by LocalContentOS under AQLIYA.
It represents a governed local content assessment and has been
reviewed and approved as documented above. It is not a
regulator-certified compliance document. All classifications
and scores are based on evidence provided and analyst review.

───────────────────────────────────────────
AI assists. Humans decide. Evidence governs.
───────────────────────────────────────────
Generated by AQLIYA LocalContentOS v0.1
═══════════════════════════════════════════
```

### Export Patterns (Reuse)

Follow AuditOS export patterns:

- `src/lib/audit/export/types.ts` — Export format types
- `src/lib/audit/export/pdf-exporter.ts` — PDF generation
- `src/lib/audit/export/xlsx-exporter.ts` — XLSX generation

Create LC-specific exporter that reuses the same libraries (pdfkit, xlsx) with LC domain data.

### Export Flow

```
1. User requests report type + format
2. System collects current project state (score, classifications, findings, evidence index)
3. System applies export template with metadata
4. System stores export file via platform storage
5. System records export to PlatformAuditLog
6. User downloads export file
```
