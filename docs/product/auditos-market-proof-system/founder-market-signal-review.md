## Purpose

هذا المستند هو إطار مراجعة دورية للمؤسس (founder) لتحليل إشارات السوق المستخلصة من أول 3-5 عملاء، وتحويلها إلى قرارات استراتيجية وتجارية ومنتجية.

## Review Cadence

| النوع | التكرار | المدة | التركيز |
|-------|---------|-------|---------|
| Weekly Signal Check | أسبوعي | 20-30 دقيقة | تحركات العملاء الحالية |
| Monthly Signal Review | شهري | 60-90 دقيقة | أنماط وتوجهات السوق |
| Quarterly Position Review | ربع سنوي | نصف يوم | قرارات positioning واستراتيجية |

## Weekly Signal Check

### أسئلة إلزامية لكل عميل حالي:

1. ما أقوى signal إيجابي ظهر هذا الأسبوع؟
2. ما الـ signal السلبي أو المتردد؟
3. هل buyer confidence ترتفع أم تنخفض؟
4. ما الـ objection الأكثر تكرارًا؟
5. هل هناك طلب لم نستطع تلبيته؟

### مخرجات إلزامية:

- [ ] تحديث health status لكل عميل
- [ ] تحديد عميل واحد يحتاج تدخل founder مباشر
- [ ] تسجيل learning واحدة على الأقل في learning log

## Monthly Signal Review

### المحور 1: Customer Value Signals

| السؤال | التحليل |
|--------|---------|
| ما أكثر فئة proof قيمة ظهرت عبر العملاء؟ | (time saved / workflow clarity / review improvement / traceability / stakeholder confidence / reduced rework) |
| ما الـ use case الأكثر إثباتًا للقيمة؟ | (أي من use cases الستة) |
| ما الـ segment الأسرع في إدراك القيمة؟ | (audit firm / accounting office / CFO team / SME finance) |
| ما الـ workflow step الأكثر إبهارًا للعملاء؟ | (trial balance mapping / statement drafting / evidence tracking / review queue / etc) |

### المحور 2: Market Objection Patterns

| السؤال | التحليل |
|--------|---------|
| ما أكثر 3 objections تكرارًا هذا الشهر؟ | |
| هل objections تتغير مع تقدم العلاقة؟ | (early objections vs late objections) |
| هل هناك objection يمنع التحول إلى paid؟ | |
| هل نحتاج تعديل في packaging أو demo بسبب objections؟ | |

### المحور 3: Conversion Signals

| السؤال | التحليل |
|--------|---------|
| كم عميلًا انتقل من pilot إلى paid؟ | |
| ما أسرع مسار للتحول إلى paid؟ | |
| ما الـ trigger الأساسي لقرار paid؟ | |
| ما الـ blocker الأساسي أمام paid؟ | |

### المحور 4: Competitive & Positioning Signals

| السؤال | التحليل |
|--------|---------|
| ما البدائل التي يذكرها العملاء؟ | (manual / spreadsheets / other tools) |
| أين يشعر العملاء أن AuditOS مختلف فعليًا؟ | |
| أين يشعر العملاء أن AuditOS غير مكتمل؟ | |
| هل positioning الحالي واضح كفاية في السوق؟ | |

### المحور 5: Referenceability Pipeline

| السؤال | التحليل |
|--------|---------|
| كم عميلًا مؤهلًا لـ Tier 1 أو Tier 2 reference؟ | |
| هل لدينا case study واحدة قابلة للنشر؟ | |
| ما مدى استعداد العملاء للمشاركة كمراجع؟ | |

## Quarterly Position Review

### الأسئلة الاستراتيجية:

1. هل الـ ICP (Ideal Customer Profile) ما زال صحيحًا أم يحتاج تعديلًا؟
2. هل positioning يحتاج إعادة ضبط بناءً على إشارات السوق؟
3. هل هناك segment أسرع في الـ adoption يستحق تركيزًا أكبر؟
4. هل الـ use cases الأكثر طلبًا تتطابق مع roadmap الحالي؟
5. هل الـ pricing model الحالي متناسب مع القيمة المثبتة؟
6. هل نحتاج إلى تغيير في demo script أو sales narrative؟
7. ما أقوى 3 إشارات إيجابية يمكن استخدامها في المحادثات الاستثمارية؟

### مخرجات إلزامية:

1. تحديث أو تأكيد ICP
2. قرار بشأن أي تعديل في positioning
3. تحديد أولويات الـ roadmap بناءً على إشارات السوق
4. توثيق الـ market signals القابلة للاستخدام في investment conversations
5. خطة الـ proof building للربع القادم

## Signal Categories

| الفئة | أمثلة |
|-------|-------|
| Strong positive | عميل انتقل إلى paid بسرعة، طلب reference تلقائي، وسّع scope بدون طلب |
| Moderate positive | عميل أكمل pilot بنجاح، value واضحة، يدرس paid |
| Neutral / uncertain | عميل متردد، value ظهرت لكن الـ commitment غير واضح |
| Concerning | عميل بطيء في الـ adoption، objections تتراكم، engagement ضعيف |
| Negative | عميل أوقف pilot، value لم تظهر، fit غير موجود |

## Founder Decision Triggers

متى يجب على المؤسس التدخل المباشر:

1. أي عميل قريب من paid conversion — يحتاج حضور founder في closeout conversation
2. أي objection متكرر عبر 3 عملاء أو أكثر — يحتاج تعديل في packaging أو demo
3. أي signal يفيد بأن positioning غير مفهوم — يحتاج إعادة صياغة فورية
4. أي عميل يطلب referenceability — يحتاج قرار founder في tier والاستخدام
5. أي pattern يشير إلى segment أو use case غير متوقع — يحتاج تحليل founder مباشر

## Rules

1. لا تبني قرارات استراتيجية على signal من عميل واحد فقط
2. لا تتجاهل signal متكرر عبر أكثر من عميل
3. فرّق بين رأي عميل واحد و pattern في السوق
4. الـ signal الحقيقي هو ما يتكرر عبر عملاء مستقلين
5. وثّق كل signal قبل اتخاذ قرار بناءً عليه
6. لا تبالغ في تفسير الإشارات المبكرة — السوق صغير جدًا
