# LocalContentOS Pilot Runbook

**Status:** Analyst-Led Pilot Operations Guide — not software implementation
**Version:** 1.0
**Phase:** Phase 2 (Manual Workflow Prototype) — maps to roadmap in Discovery Pack
**Depends on:** `docs/product/localcontentos-discovery-pack/`, `docs/product/localcontentos-data-templates/`

---

## Purpose

This runbook converts the LocalContentOS Discovery Pack and Data Templates Pack into a **complete analyst-led pilot operating procedure**. It defines:

- How to identify, qualify, and onboard a pilot customer
- Step-by-step operating procedure for the pilot analyst
- Templates for pilot meetings, decisions, and commercial follow-up
- Risk register and success criteria scorecard

**The pilot is a service engagement — not a software implementation.** No code, schema, routes, or UI exist. The analyst uses templates, review procedures, and evidence-based workflow to produce a local content assessment.

## Pilot Flow Summary

```
Discovery Call → Kickoff → Data Request → Data Receipt → Intake Review
→ Vendor Review → Spend Review → Contract Review → Evidence Review
→ Classification → Exceptions → Findings → Management Review
→ Pilot Summary → Decision Meeting → Commercial Follow-up
```

**Duration:** 6–8 weeks
**Scope:** One reporting period, one procurement category, sample suppliers/contracts

## Regulatory Boundary

This runbook is designed for an **internal local content governance assessment**. The pilot does not:

- Constitute an official regulatory submission
- Replace statutory local content reporting
- Represent LCGPA or any regulatory body's endorsement
- Provide binding legal or compliance determinations
- Include any software or technology implementation

All classification criteria and findings are based on customer-provided data and evidence. They are reviewed by humans and presented as draft management materials, not final regulatory determinations.
