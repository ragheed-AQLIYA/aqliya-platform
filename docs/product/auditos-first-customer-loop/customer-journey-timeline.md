# AuditOS Customer Journey Timeline

Source of truth:

1. `docs/product/auditos-outbound-kit/`
2. `docs/product/auditos-live-pilot-management/`

## Purpose

هذا المستند يحدد timeline بسيطة يمكن تكرارها لأول العملاء من demo إلى paid / reference decision.

## Standard Journey

```text
Demo
    → Discovery refinement
    → Pilot agreement
    → Onboarding
    → Active pilot
    → Evidence capture
    → Success review
    → Post-pilot decision
    → Paid / Extend / No-Go
    → Reference / Learning capture
```

## Typical Timeline

### Week 0

1. demo completed
2. fit confirmed
3. use case agreed

### Week 1

1. onboarding
2. owner alignment
3. success criteria locked

### Week 2-4

1. active pilot execution
2. communication log updates
3. evidence capture
4. risk monitoring

### Week 4-6

1. success review
2. post-pilot memo
3. decision discussion

### Week 6+

1. paid transition
2. pilot extension
3. no-go closeout
4. case-study capture if relevant

## Timeline Rule

إذا بقي العميل عالقًا بين مرحلتين بدون owner أو next action، فهو خارج loop التشغيلية الفعالة.
