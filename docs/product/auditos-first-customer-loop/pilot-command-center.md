## Purpose

هذا المستند هو لوحة القيادة التشغيلية العليا لإدارة أول 3-5 عملاء pilot بشكل مركزي.

## Command Center Snapshot

لكل عميل سجل:

1. Account name
2. Segment
3. Internal owner
4. Current phase
5. Health status
6. Biggest risk
7. Biggest proof signal
8. Next decision date
9. Expected outcome: paid / extend / no-go / reference candidate

## Recommended Health Labels

1. `green` — pilot تتحرك، stakeholder engagement جيد، evidence تتجمع
2. `yellow` — progress موجود لكن هناك risk أو بطء
3. `red` — pilot مهددة أو لا تتقدم بشكل كافٍ

## Weekly Command Questions

1. من أقرب عميل للتحول إلى paid؟
2. من أقرب عميل للتحول إلى reference؟
3. من العميل الذي يعطينا أقوى learning؟
4. أين يوجد خطر حقيقي على نجاح الـ pilot؟
5. ما القرار الذي يجب أخذه هذا الأسبوع؟

## Command Rule

إذا لم يكن لكل pilot:

1. owner واضح
2. health status واضح
3. next action واضح
4. next decision date واضحة

فهي ليست تحت control فعلي.
