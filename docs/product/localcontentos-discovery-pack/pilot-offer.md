# LocalContentOS — Pilot Offer

**Status:** Discovery / Planned (not implemented)
**Version:** 1.0 — Discovery Pack
**Note:** This is a pilot offer design document. No pilot is active. This describes what a pilot would include when the product is ready.

---

## Pilot Overview

| Item           | Detail                                                                                         |
| -------------- | ---------------------------------------------------------------------------------------------- |
| **Product**    | LocalContentOS                                                                                 |
| **Status**     | Discovery / Planned                                                                            |
| **Pilot Type** | Paid pilot (discounted) or proof-of-concept (free, time-boxed)                                 |
| **Duration**   | 6–8 weeks                                                                                      |
| **Scope**      | One reporting period, one procurement category or business unit, sample of suppliers/contracts |

---

## Pilot Objective

Validate that LocalContentOS can:

1. Accept real customer procurement and supplier data
2. Classify spend and suppliers by local content criteria
3. Support evidence-based local content review and exception handling
4. Generate a management-ready local content report
5. Provide a clear audit trail of all local content decisions

The pilot is not a production implementation. It is a structured evaluation of the product concept and workflow.

---

## What Customer Provides

| Item                            | Detail                                            | Format          |
| ------------------------------- | ------------------------------------------------- | --------------- |
| Vendor master list              | All vendors relevant to the pilot scope           | CSV / XLSX      |
| Procurement spend report        | Transactions for the pilot period and scope       | CSV / XLSX      |
| Purchase orders                 | POs linked to the pilot transactions              | CSV / XLSX      |
| Supplier classification         | Customer's current local/non-local classification | CSV / XLSX      |
| Supporting evidence             | CRs, invoices, certificates (sample)              | PDF             |
| Subject matter expert           | Local content officer or procurement analyst      | Time commitment |
| Access to relevant data systems | ERP extracts, supplier database (read-only)       | As needed       |

---

## What AQLIYA Provides

| Item                     | Detail                                                                  |
| ------------------------ | ----------------------------------------------------------------------- |
| **Consultant / Analyst** | LocalContentOS product analyst to guide the pilot                       |
| **Data processing**      | Import and normalize customer data (manual or tool-assisted)            |
| **Classification setup** | Configure classification criteria based on customer policy              |
| **Evidence review**      | Structured evidence review and gap analysis                             |
| **Exception handling**   | Document and present classification exceptions                          |
| **Report generation**    | Draft local content report with findings and recommendations            |
| **Pilot report**         | Final pilot report — methodology, findings, recommendations, next steps |
| **Executive summary**    | Management-ready summary of pilot outcomes                              |

**Note:** The pilot is delivered as a **service engagement** (analyst-led), not as a software application. There is no LocalContentOS software for the customer to log into. The workflow is executed by the AQLIYA team using tools and templates, with the customer providing data and expertise.

---

## What Is Excluded

- Software access or licenses (product is not implemented)
- Technical integration with customer ERP, HR, or other systems
- Regulatory filing or submission
- Official local content certification or audit
- Legal advice or compliance guarantees
- Training (beyond pilot scope orientation)
- Production deployment or ongoing support

---

## Success Criteria

| Criterion                                            | How Measured                            |
| ---------------------------------------------------- | --------------------------------------- |
| Customer data successfully imported and normalized   | Data acceptance rate > 90%              |
| Suppliers classified by locality                     | 100% of pilot suppliers classified      |
| Spend classified by local/non-local                  | 100% of pilot transactions classified   |
| Evidence linked to classifications                   | At least one evidence item per supplier |
| Exceptions identified and documented                 | Exception log completed                 |
| Findings and gaps report delivered                   | Report accepted by customer             |
| Management-ready summary delivered                   | Summary accepted by customer            |
| Customer can make informed decision about next steps | Post-pilot decision meeting held        |

## Decision After Pilot

After the pilot, the customer and AQLIYA decide jointly:

| Option                            | Description                                                                    |
| --------------------------------- | ------------------------------------------------------------------------------ |
| **Proceed to MVP Implementation** | Customer commits to building LocalContentOS (funded implementation project)    |
| **Extend Pilot**                  | Expand scope (more categories, longer period, more data) before final decision |
| **Wait and Revisit**              | Customer not ready yet — revisit after 6 months or when conditions change      |
| **Not Proceed**                   | Product is not the right fit for this customer                                 |
