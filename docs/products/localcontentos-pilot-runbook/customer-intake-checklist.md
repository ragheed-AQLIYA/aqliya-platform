## Intake Acceptance Criteria

All of the following must be met to proceed:

- [ ] Customer identity verified and contact confirmed
- [ ] Pilot scope clearly defined and agreed
- [ ] Required files identified and commitment to provide them received
- [ ] Data owner and approval owner identified
- [ ] Data sensitivity classification agreed
- [ ] Pilot duration and meeting cadence accepted
- [ ] Commercial terms agreed (paid or free)
- [ ] Regulatory boundary acknowledged and accepted
- [ ] Evidence folder structure agreed

---

## Intake Rejection / Hold Criteria

Any of the following triggers a hold or rejection:

| Criterion                                       | Action                                                  |
| ----------------------------------------------- | ------------------------------------------------------- |
| Customer cannot identify data owner             | Hold until owner is assigned                            |
| Customer expects regulatory submission          | Educate on scope — reject if expectation persists       |
| Customer unwilling to provide any required data | Hold — define minimum viable dataset                    |
| Data sensitivity prevents reasonable analysis   | Reject — scope or security cannot be satisfied          |
| No decision maker identified                    | Hold — no one can approve pilot participation           |
| Pilot scope is undefined or too broad           | Hold — scope must be narrowed before proceeding         |
| Customer expects software or platform access    | Educate — this is an analyst-led pilot with no software |

---

## Intake Outcome

| Decision     | Next Step                                                                  |
| ------------ | -------------------------------------------------------------------------- |
| **Accepted** | Proceed to kickoff meeting. Send customer instructions and data templates. |
| **Hold**     | Document conditions to resolve. Set follow-up date.                        |
| **Rejected** | Document reason. Archive for future reference.                             |
