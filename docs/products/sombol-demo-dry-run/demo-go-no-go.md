## GO Criteria

يجب أن تتوفر جميع الشروط التالية لإصدار قرار GO:

| # | الشرط | متوفر؟ |
|---|---|---|
| 1 | جميع مسارات العمل تعمل (انظر environment checklist) | ☐ |
| 2 | جميع البيانات التجريبية جاهزة وآمنة (انظر test data plan) | ☐ |
| 3 | تم إجراء تمرين كامل (dry-run) مرة واحدة على الأقل | ☐ |
| 4 | جميع المشكلات الحرجة (🔴) مغلقة | ☐ |
| 5 | جميع المشكلات العالية (🟡) مغلقة أو معروفة ومعترف بها | ☐ |
| 6 | خطة بديلة جاهزة حال تعطل الإنترنت أو المنصة | ☐ |
| 7 | فريق AQLIYA جاهز (فني + تجاري) | ☐ |
| 8 | ملفات العرض التجريبية محفوظة محلياً (fallback) | ☐ |

**GO:** ✅ كل الشروط متوفرة → العرض جاهز.

---

## CONDITIONAL GO Criteria

إذا كانت بعض الشروط غير متوفرة ولكن يمكن تجاوزها:

| # | الشرط | متوفر؟ |
|---|---|---|
| 1 | جميع المشكلات الحرجة (🔴) مغلقة | ☐ |
| 2 | المشكلات العالية (🟡) معروفة ويمكن تجنبها في العرض | ☐ |
| 3 | لدى مقدم العرض خطة بديلة لكل مشكلة معروفة | ☐ |
| 4 | العميل على دراية بأنها مرحلة MVP | ☐ |

**CONDITIONAL GO:** ✅ الشروط أعلاه متوفرة → العرض جائز مع تحفظات.

---

## NO-GO Criteria

أي من الشروط التالية يمنع العرض:

| # | الشرط | موجود؟ |
|---|---|---|
| 1 | مشكلة حرجة (🔴) غير محلولة — تمنع إحدى الخطوات الأساسية | ☐ |
| 2 | العرض لم يتم التمرّن عليه مطلقاً | ☐ |
| 3 | بيانات حقيقية للعملاء في بيئة العرض | ☐ |
| 4 | الإنترنت أو المنصة غير مستقرة بشكل متكرر | ☐ |
| 5 | فريق AQLIYA غير جاهز | ☐ |

**NO-GO:** ❌ أي شرط أعلاه موجود → العرض ملغي أو مؤجل.
