1. `docs/products/auditos-first-customer-loop/referenceability-decision-template.md`

## Purpose

هذا المستند ينظم كيفية طلب reference أو quote أو case-study permission من العميل.

## Referenceability Paths

1. public quote
2. anonymized case study
3. private reference call
4. metric-only proof
5. no reference

## When To Ask

اطلب reference فقط إذا:

1. value ظهرت بوضوح
2. stakeholder satisfaction جيدة
3. هناك evidence قابلة للاستخدام

## Public Quote Request

اطلبه عندما:

1. العميل مرتاح علنًا
2. يوجد stakeholder مناسب
3. الرسالة واضحة وقابلة للاقتباس

## Anonymized Case Study Request

اطلبه عندما:

1. القيمة واضحة
2. العميل لا يريد الكشف عن الاسم
3. يمكن مشاركة workflow/value بشكل anonymized

## Private Reference Call

استخدمه عندما:

1. buyer satisfied
2. لا يريد public exposure
3. لكنه يقبل الحديث مع prospects لاحقًا

## Metric-Only Proof

استخدمه عندما:

1. لا توجد موافقة على quote أو call
2. لكن توجد نتائج أو signals كمية قابلة للمشاركة بدون كشف العميل

## What To Ask For

1. quote short
2. permission to mention account type
3. permission for anonymized case study
4. permission for private reference call

## What Not To Do

1. لا تطلب reference قبل أن يقتنع العميل داخليًا بالقيمة
2. لا تضغط على stakeholder متحفظة
3. لا تعد باستخدام الاسم علنًا قبل موافقة صريحة
