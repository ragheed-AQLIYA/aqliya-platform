# Demo and Pilot Claims Script — AQLIYA

## Purpose

Provide a precise script of what may and may not be said during a Controlled Customer Demo or Controlled Pilot. Prevents overclaims and protects commercial truthfulness.

## When to Use

Before every customer-facing demo and pilot kickoff meeting. Must be reviewed by product lead.

## Prohibited Claims

| ❌ Do NOT Say                       | Why                                                  |
| ----------------------------------- | ---------------------------------------------------- |
| "Production-ready"                  | Not deployed in production. P2 security items open.  |
| "Enterprise-hardened at scale"      | Scale not tested. No penetration test.               |
| "On-Premises proven"                | Architecture direction only. Not deployed.           |
| "Air-Gapped supported"              | Not implemented.                                     |
| "Fully autonomous AI"               | AI always has human review. No autonomous decisions. |
| "SSO / Active Directory ready"      | Not implemented.                                     |
| "Certified / regulator-approved"    | No certification obtained.                           |
| "SalesOS is a complete product"     | Prototype only.                                      |
| "LocalContentOS is paid production" | Pilot-ready بشروط only.                              |

## Demo Guardrails

1. **No real customer data** in demo environment — use seed/mock data only.
2. **Demo disclaimer** must be visible: "بيئة عرض توضيحي — بيانات غير حقيقية" / "Demo environment — not real data".
3. **Do not navigate** to unbuilt routes or click "coming soon" buttons.
4. **If asked about missing features**: say "ضمن خطتنا المستقبلية" / "on our roadmap".

## Pilot Guardrails

1. **Pilot scope must be signed** before any customer data enters the system.
2. **No paid commitments** discussed until pilot success criteria are evaluated.
3. **Weekly status reports** during pilot with clear pass/fail indicators.
4. **Stop conditions** reviewed before each pilot milestone.

## Owner

- **Claims Gatekeeper:** AQLIYA Product Lead
- **Review cycle:** Before every demo/pilot meeting with a new customer
