## How to Avoid Underpricing

| Risk                                                   | Mitigation                                                          |
| ------------------------------------------------------ | ------------------------------------------------------------------- |
| Customer sees pilot as "just filling templates"        | Price reflects analyst expertise, methodology, governance framework |
| Customer compares to free AI chat tools                | Position as governed institutional process, not AI trick            |
| Customer expects full enterprise scope for pilot price | Define scope clearly. Charge for scope expansions.                  |
| Customer says budget is tight                          | Offer Diagnostic Review path instead of discounting                 |
| Pilot scope creeps mid-engagement                      | Track scope changes. Separate invoice for significant additions.    |

---

## When to Reject a Pilot

| Situation                              | Action                                                            |
| -------------------------------------- | ----------------------------------------------------------------- |
| Customer expects regulatory submission | Educate. If persists, reject. Risk of expectation mismatch.       |
| Customer has no data owner             | Reject. Pilot cannot succeed without data engagement.             |
| Customer has no decision owner         | Reject. No one to decide after pilot.                             |
| Scope is undefined or unrealistic      | Reject. Require clear scope definition before proceeding.         |
| Customer cannot provide any data       | Reject. No data = no analysis.                                    |
| Timeline is unreasonable (< 3 weeks)   | Reject or recommend Diagnostic Review.                            |
| Budget far below minimum viable        | Reject. Underpriced pilots set wrong expectations for both sides. |
