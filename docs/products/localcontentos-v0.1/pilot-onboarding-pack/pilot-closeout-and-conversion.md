# Pilot Closeout & Conversion — LocalContentOS v0.1

## Closeout Meeting

**Purpose:** Review pilot results, determine next steps, capture feedback.

**Duration:** 60 minutes

**Attendees:** AQLIYA pilot lead, customer sponsor, customer stakeholders

**Agenda:**

| Time   | Topic                 | Description                                                 |
| ------ | --------------------- | ----------------------------------------------------------- |
| 10 min | Pilot results review  | Walk through success criteria — which passed, which did not |
| 10 min | Customer feedback     | What worked, what didn't, what is missing                   |
| 10 min | Enhancement backlog   | Document customer-requested features and improvements       |
| 15 min | Conversion discussion | Present options for next phase                              |
| 10 min | Timeline discussion   | If converting, agree on timeline and scope                  |
| 5 min  | Next steps            | Action items, ownership, dates                              |

## Success Criteria Summary

| Status                                 | Meaning                                                     |
| -------------------------------------- | ----------------------------------------------------------- |
| All 10 measurable criteria passed      | Pilot is a technical success                                |
| Some criteria not met                  | Document gaps, determine if scope or execution issue        |
| Customer engaged and understands value | Commercial success — customer can articulate platform value |
| Customer sees next step                | Conversion-ready                                            |

## Conversion Paths

| Path                        | Description                                        | Typical Next Step                                       |
| --------------------------- | -------------------------------------------------- | ------------------------------------------------------- |
| **Paid pilot extension**    | Continue with broader scope or additional data     | Extend by 2-4 weeks with agreed success criteria        |
| **Production pilot**        | Begin transitioning to production use              | Scoped implementation plan, 4-8 week timeline           |
| **Full subscription**       | Customer commits to ongoing use                    | Annual subscription with defined user count and support |
| **Custom implementation**   | Customer needs modifications or additions          | Statement of work for custom development                |
| **Enhancement + re-engage** | Customer needs a missing feature before committing | Add feature to backlog, schedule re-engagement          |

## No-Conversion Options

| Option                 | Description                                                                                     |
| ---------------------- | ----------------------------------------------------------------------------------------------- |
| **Hold**               | Customer needs more time or internal alignment. Agree on re-engagement date (3-6 months).       |
| **No conversion**      | Customer declines to proceed. Document reasons, ask for testimonial if experience was positive. |
| **Reference use only** | Customer is willing to be a reference case study even without commercial conversion.            |

## Enhancement Backlog

Capture every feature request during closeout in a structured backlog:

| Priority | Request                                        | Customer Need | Proposed Timeline |
| -------- | ---------------------------------------------- | ------------- | ----------------- |
| P0       | Blocking — without this, customer cannot adopt | Critical      | Immediate         |
| P1       | Important — significant value addition         | High          | Next release      |
| P2       | Nice to have — improves experience             | Medium        | Future            |
| P3       | Future — strategic direction                   | Low           | Roadmap           |

## Customer Reference / Case Study

If the customer had a positive experience, discuss:

- Permission to use customer name and logo
- Case study or success story publication
- Joint presentation or webinar
- Testimonial or quote
- Reference calls for prospective customers

## Data Retention

At pilot closeout:

1. Export all customer data and provide to customer
2. Agree on data retention period (default: 30 days post-closeout)
3. Delete customer data from pilot environment after retention period
4. Provide written confirmation of data deletion

## Closeout Documentation

Generate and share with customer:

- Pilot results summary (criteria met / not met)
- Enhancement backlog
- Conversion proposal
- Data export package
- Next steps agreement
