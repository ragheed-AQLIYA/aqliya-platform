# AuditOS — Controlled Client Demo Agenda

## Overview

**Duration:** 60 minutes  
**Format:** Live product walkthrough with Q&A  
**Audience:** Client stakeholders (audit team, IT, management)  
**Presenter:** AQLIYA Product Team  
**Environment:** Local development server with seeded demo data

---

## Agenda

| Time | Duration | Section                             | Activity                                                                                              |
| ---- | -------- | ----------------------------------- | ----------------------------------------------------------------------------------------------------- |
| 0:00 | 5 min    | **1. Introduction and Positioning** | Welcome, what AuditOS is, who it is for, core principle                                               |
| 0:05 | 5 min    | **2. Problem Framing**              | Current audit challenges: fragmented tools, weak traceability, manual drafting, rubber-stamp approval |
| 0:10 | 5 min    | **3. Product Overview**             | Dashboard, engagement list, workflow progress, AI Outputs panel                                       |
| 0:15 | 20 min   | **4. Product Walkthrough**          | Step-by-step journey through the full audit workflow (see details below)                              |
| 0:35 | 5 min    | **5. AI Capabilities**              | AI as assistant-only: draft notes, evidence suggestions, analytical review                            |
| 0:40 | 3 min    | **6. Traceability and Audit Trail** | TraceabilityDrawer demonstration, audit event timeline                                                |
| 0:43 | 3 min    | **7. Approval Controls**            | Readiness gate, blocked approval scenario, partner approval with rationale                            |
| 0:46 | 3 min    | **8. Export/Reporting**             | JSON export, bilingual labels, draft/approved status markers                                          |
| 0:49 | 5 min    | **9. Known Limitations**            | Honest disclosure: JSON-only, no virus scanning, production auth status                               |
| 0:54 | 5 min    | **10. Pilot Scope Discussion**      | What is included, what is excluded, timeline, success criteria                                        |
| 0:59 | 1 min    | **11. Next Steps**                  | Feedback collection, production roadmap, sign-off process                                             |

## Materials Needed

- Demo environment running with seeded data
- Projector or screen sharing
- Demo script printed or on second monitor
- Known limitations summary ready
- Export files pre-generated for backup if live demo fails
- FAQ document for reference during Q&A

---

## Backup Plan

If the live demo fails (server down, data corrupted, network issue):

1. Have pre-recorded screenshots of each workflow step
2. Have pre-generated export JSON files ready to share
3. Continue with walkthrough using screenshots
4. Schedule a second demo if needed
