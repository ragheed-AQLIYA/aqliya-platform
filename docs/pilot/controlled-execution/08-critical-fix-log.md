**Document:** 08-critical-fix-log.md  
**Purpose:** Log all post-freeze fixes that are approved during the pilot window.  

---

## Fix Policy

After freeze, only the following fix types are allowed:

- **Critical pilot blocker** — Blocks TB processing or demo
- **Broken route/link** — Causes 404 during customer demo
- **Broken UI visibility** — Outputs unreadable or inaccessible
- **Security concern** — Data exposure or auth bypass
- **Data loss concern** — Engagement data at risk
- **Validation failure** — Build/type/health check fails

These fix types are **forbidden** after freeze:

- New feature
- New module
- Architecture refactor
- Visual redesign
- Experimental AI
