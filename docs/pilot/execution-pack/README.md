## Purpose

This pack contains all operational documents needed to run the first real customer pilot of AuditOS. Use these documents sequentially as the customer moves through the pilot lifecycle.

## When to Use

Open this pack **after**:

- Customer has agreed to participate in the pilot
- AQLIYA team has received the customer's Trial Balance file
- The engagement has been created in the AuditOS workspace

## Pilot Sequence

```
1. Receive TB file          → Customer sends file
2. Run intake checklist      → 01-trial-balance-intake-checklist.md
3. Create engagement         → AuditOS workspace
4. Upload TB                 → Trial Balance upload page
5. Run onboarding checklist  → 02-customer-onboarding-checklist.md
6. Run demo walkthrough      → 03-demo-walkthrough-script.md
7. Run reviewer QA           → 04-reviewer-qa-checklist.md
8. Run financial QA          → 05-financial-output-qa-checklist.md
9. Run traceability QA       → 06-traceability-qa-checklist.md
10. Capture feedback          → 07-pilot-feedback-form.md
11. Escalate issues           → 08-issue-escalation-workflow.md (as needed)
12. Evaluate success          → 09-pilot-success-criteria.md
13. Conduct review            → 10-post-pilot-review-memo.md
```

## Required Inputs

Before starting the pilot, ensure the following are ready:

- [ ] Customer Trial Balance file (Excel or CSV)
- [ ] Customer entity details (name, reporting period, currency)
- [ ] Customer reviewer contact information
- [ ] AQLIYA reviewer assigned
- [ ] Engagement created in AuditOS workspace
- [ ] Pilot lead assigned
- [ ] Confidentiality agreement in place
- [ ] Pilot timeline agreed with customer

## Completion Criteria

The pilot is considered complete when:

- All QA checklists have been run and signed off
- Customer feedback has been collected
- Issues have been documented and escalated if needed
- Success criteria have been evaluated
- Post-pilot review memo has been written

## Next Step After Pilot

- If **Go** → Begin commercial readiness preparation
- If **Conditional Go** → Address blockers, re-run affected checklists
- If **No-Go** → Document gaps, schedule internal review, determine next actions
