# Human Review Requirement

## Purpose

This document defines the mandatory human review requirements for all AuditOS AI-generated outputs. It operationalizes the core doctrine: **AI assists. Humans decide. Evidence governs.**

## General Principle

Every output generated by AI in AuditOS is a draft. No AI-generated output may be presented as final, approved, or authoritative without explicit human review and acceptance.

## Outputs Requiring Human Review

| Output | AI Role | Human Action Required | Before |
|--------|---------|----------------------|--------|
| Account mapping suggestion | Propose mapping candidates | Confirm or correct mapping | Mapping is applied |
| Trial balance observation | Flag anomalies | Review and disposition | Observation is included in findings |
| Financial statement draft | Generate from mapped data | Review each statement line | Statement is included in package |
| Notes draft | Generate from available data | Review, supplement, approve | Note is included in package |
| Evidence requirement | Identify needed evidence | Confirm requirements | Evidence request is sent |
| Review finding | Identify potential issue | Accept, modify, or reject | Finding is included in report |
| Recommendation | Draft suggested language | Review, edit, approve | Recommendation is issued |

## Review Levels by Output Type

### Account Mapping

| Confidence | Review Required | Reviewer |
|------------|----------------|----------|
| High (85%+) | Recommended | Operator |
| Medium (60-84%) | Required | Operator |
| Low (below 60%) | Required | Senior Reviewer |
| Unmapped | Manual entry | Senior Reviewer |

### Financial Statements

| Statement | Review Required | Reviewer |
|-----------|----------------|----------|
| Statement of Financial Position | Required | Reviewer |
| Statement of Profit or Loss | Required | Reviewer |
| Statement of Changes in Equity | Required (if generated) | Reviewer |
| Statement of Cash Flows | Required (if generated) | Reviewer |

### Notes

| Note Type | Review Required | Reviewer |
|-----------|----------------|----------|
| Populated from trial balance | Required | Reviewer |
| Partially populated | Required + supplement | Reviewer |
| Placeholder | Required + provide data | Reviewer |
| Policy template | Required | Reviewer |

### Findings

| Severity | Review Required | Reviewer |
|----------|----------------|----------|
| Critical | Required | Partner |
| High | Required | Manager |
| Medium | Required | Reviewer |
| Low | Recommended | Reviewer |

## Review Actions

For every reviewable item, the reviewer may:

| Action | Description | Recorded |
|--------|-------------|----------|
| **Accept** | Confirm AI-generated output as-is | Reviewer identity, timestamp |
| **Modify** | Edit AI-generated output before accepting | Reviewer identity, changes, rationale, timestamp |
| **Reject** | Decline AI-generated output with reason | Reviewer identity, rationale, timestamp |
| **Escalate** | Send to higher authority for review | Reviewer identity, escalation reason, timestamp |

## Prohibited Patterns

| Pattern | Why | Alternative |
|---------|-----|-------------|
| Auto-accept AI outputs | Removes human accountability | Default to pending review |
| Bulk accept without review | Circumvents governance | Require item-level action |
| AI finalizes outputs | AI cannot issue conclusions | Human must take final action |
| Reviewer accepts without viewing evidence | Undermines evidence doctrine | Require evidence view before accept |
| Configurable bypass of review | Governance must be structural | Review gates are immutable |

## Enforcement

The workflow engine enforces human review requirements structurally:

```txt
- A finding with severity Critical cannot advance to publication without Partner approval
- A note flagged as missing critical data cannot be included as complete
- An account mapping with Low confidence cannot be applied without Senior Reviewer confirmation
- Every review action is recorded as an attributable governance event
- No output enters the publication package without all required reviews completed
```
