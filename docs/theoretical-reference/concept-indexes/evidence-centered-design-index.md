# Evidence-Centered Design

## Definition
Evidence-Centered Design means that every system output — recommendation, finding, signal, report — must be reducible to traceable, verifiable evidence. Evidence is not metadata; it is the core data type.

## Why It Matters
It transforms trust from an abstract claim into a verifiable chain. Every assertion the system makes is backed by evidence the reviewer can inspect, challenge, and accept or reject, making audit workflows transparent and defensible.

## Governance Implications
Evidence state lifecycle is a governance mechanism (candidate → verified → accepted). Each transition requires a defined authority and is logged immutably, ensuring no evidence enters a report without passing through governed gates.

## AuditOS Relevance
Evidence is the unit of trust in every audit workflow. AuditOS is built around evidence as a first-class entity — every finding is an assembly of evidence objects, and every conclusion is a judgment on that evidence.

## Notes for Future Expansion
Evidence schemas may become domain-specific (e.g., evidence standards for financial vs. compliance audits), but the core lifecycle should remain universal.
