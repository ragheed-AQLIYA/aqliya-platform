# Principle-to-Product Traceability

## Principles as root of all system behavior

### Principle: Evidence is the unit of trust

- **Doctrine Source:** Part 02 — Trust is derived from verifiable evidence, not from identity or role
- **Product Rule:** Every action that mutates system state MUST produce a corresponding evidence record before the transaction commits
- **MVP Feature:** Transaction wrapper that requires evidence attachment for all write operations
- **Verification:** No write operation succeeds without an attached evidence record in the audit log

### Principle: Governance is structural

- **Doctrine Source:** Part 01, Part 04 — Governance rules are encoded in the system topology, not in policy documents
- **Product Rule:** Each governance gate is a node in the workflow graph; bypassing a gate requires a structural change, not a configuration flag
- **MVP Feature:** Workflow engine routes transitions through gate nodes that enforce preconditions
- **Verification:** Every state transition passes through at least one gate node validated by the workflow engine

### Principle: AI assists. Humans decide.

- **Doctrine Source:** Part 03 — AI produces recommendations; only humans authorise binding actions
- **Product Rule:** All recommendations generated by AI MUST be reviewed and explicitly approved by a human actor before they take effect
- **MVP Feature:** Approval queue that surfaces AI recommendations and blocks execution until human sign-off
- **Verification:** The audit log records both the AI recommendation and the distinct human approval event for every binding action

### Principle: Workflow before dashboard

- **Doctrine Source:** Part 05 — Dashboards are read-only projections of workflow state, not systems of record
- **Product Rule:** All data displayed on dashboards MUST originate from workflow events stored in the event log
- **MVP Feature:** Read-only query layer that sources all dashboard data from the event store
- **Verification:** Dashboard views contain zero write paths; all mutations are routed through the workflow engine
