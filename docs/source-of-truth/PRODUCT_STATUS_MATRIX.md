## Reality Notes

- Office AI Assistant is real code and real data-backed workflow, but it remains a shared governed application rather than a standalone product claim.
- Sunbul is real and usable, but it is best positioned as a custom/client-specific workspace.
- workflowos is real in code but currently duplicates Sunbul implementation patterns and should remain internal/custom in framing.
- SalesOS is mock-only: no Prisma, no server actions, no mutations, no persistence. It is a prototype layout with hardcoded data arrays.
- `/organizations` is mock-only: hardcoded data in `mockOrganizations` and `mockOrg` constants. Self-labels as internal prototype with amber warning banners.
- DecisionOS Phase 4 stabilization (2026-05-25): loading.tsx, error.tsx, not-found.tsx added to `/decisions/[id]/`. No business logic or auth changes.
- Generic `/settings` main page is a client-side-only local-state shell (L2). The sub-routes (`/settings/workspaces`, `/settings/platform-organization`, `/settings/audit-logs`) are real admin surfaces backed by Prisma (L4). This mixed classification must be maintained.
- LocalContentOS is implemented as L5 pilot-ready with conditions / usable v0.1 (not marketing-only or unimplemented). Workspace routes, server actions, seed data, bilingual UI, evidence upload, review/approval, exports, and audit trail exist. Binary PDF/XLSX exports implemented (2026-05-25) using existing pdfkit + xlsx libraries. Export unit tests added: PDF starts `%PDF-`, XLSX starts `PK`. Workspace now has loading/error/not-found resilience files. Mutation feedback loop verified (2026-05-23): `revalidatePath` on writes + client refresh; focused finding create PASS on seeded project. CLI: `prisma generate`, `tsc`, `lint`, `build`, local-content tests (30 tests). Must not be claimed as L6 production-hardened. Arabic PDF font rendering is P2 quality gap. Dead export server action removed. See `docs/reports/localcontentos-v0.1-documentation-truth-sync-2026-05-23.md`.
