## Reality Notes

- Office AI Assistant is real code and real data-backed workflow, but it remains a shared governed application rather than a standalone product claim.
- Sunbul is real and usable, but it is best positioned as a custom/client-specific workspace.
- workflowos is real in code but currently duplicates Sunbul implementation patterns and should remain internal/custom in framing.
- SalesOS, `/organizations`, and generic `/settings` are intentionally present but are not v0.1-complete operational modules.
- LocalContentOS remains strategic and must not be sold as implemented until workspace, persistence, governance flow, and validation evidence exist.
